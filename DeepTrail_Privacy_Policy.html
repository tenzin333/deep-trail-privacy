DeepTrail — Privacy Policy

Effective date: April 15, 2026  |  Last updated: April 15, 2026

Summary: DeepTrail is a Chrome extension that helps you build a personal knowledge graph from web pages you choose to save. Your page content is sent to our backend server for AI processing (embedding and summarization), stored in our database, and never sold, shared for advertising, or used for any purpose beyond providing DeepTrail's core functionality. You can view, delete, or request export of your data at any time.

2. What Data We Collect

DeepTrail collects data only when you actively use the extension to save a page or perform a search. We categorize collected data as follows:

2a. Data collected when you save a page

  • Web page content (text): The text content extracted from web pages you explicitly choose to save to your knowledge graph. This is classified as "website content" under Chrome Web Store data types.
  • Page metadata: The URL, page title, and timestamp of each saved page. Used for node identification, deduplication, and display.

2b. Data collected when you search

  • Search queries: Text queries you type into DeepTrail's search bar are sent to our backend to perform semantic search across your knowledge graph.

2c. Data collected for authentication

  • Email address: If you create an account, we store your email address to identify your account.
  • Authentication token: A session token stored locally via Chrome's chrome.storage API to keep you signed in.

2d. Data stored locally on your device

  • Authentication token: Stored in chrome.storage.local.
  • Extension settings/preferences: Any user preferences are stored locally via chrome.storage.

3. What Data We Do NOT Collect

  • We do not collect your general browsing history. We only process pages you explicitly save.
  • We do not collect passwords, form data, financial information, or health information.
  • We do not track your activity across websites.
  • We do not collect data when the extension is inactive or when you are not actively using a DeepTrail feature.
  • We do not collect personally identifiable information beyond your email address (if you create an account).
  • We do not collect your IP address for tracking purposes (though our hosting provider may log IP addresses in standard server logs as described in Section 6).
  • We do not use cookies or any browser-based tracking technologies.
  • We do not run any analytics or telemetry services.

4. How We Use Your Data

Your data is used exclusively to provide DeepTrail's core functionality — building and querying your personal knowledge graph. Specifically:

Data type Purpose Processing location
Web page content Generate semantic embeddings for similarity search; generate summaries for knowledge graph nodes Sent to Gemini API (embeddings) and Groq API (summarization), then stored on our database
Page metadata (URL, title) Node identification, deduplication, display in knowledge graph Stored on our database
Search queries Semantic search across your knowledge graph Sent to Gemini API (query embedding), compared against stored embeddings on our database
Email address Account identification and authentication Stored on our database
Authentication token Keep you signed in across sessions Stored locally on your device via chrome.storage

We do not use your data for any other purpose, including advertising, profiling, analytics, marketing, or training AI models.

6. Data Storage, Retention, and Security

6a. Where your data is stored

  • Server-side: Your knowledge graph data (nodes, edges, embeddings, metadata, account info) is stored in a PostgreSQL database hosted on Neon.tech (cloud infrastructure). Our backend API runs on Render.
  • Client-side (your device): Authentication tokens and extension settings are stored locally via Chrome's chrome.storage API.

6b. How long we keep your data

  • Your data is retained for as long as you have an active DeepTrail account.
  • If you request deletion, all your data will be permanently removed from our servers within 30 days.
  • If you uninstall the extension, your server-side data is retained so you can reinstall and recover your knowledge graph. To delete server-side data, you must explicitly request deletion (see Section 8).
  • Locally stored data (authentication tokens, preferences) is removed automatically when you uninstall the extension.
  • Standard server logs (which may include IP addresses) generated by our hosting provider are retained per their default retention policies and are not used by us for any purpose.

6c. How we protect your data

  • All data transmission between the extension and our backend uses HTTPS/TLS encryption.
  • All data transmission between our backend and third-party APIs (Gemini, Groq) uses HTTPS/TLS encryption.
  • Authentication tokens are stored securely using Chrome's chrome.storage API.
  • Each user's data is isolated — you cannot access another user's knowledge graph.
  • Database access is restricted to authenticated API requests only.

7. Permissions Justification

DeepTrail requests the following Chrome extension permissions. Each is required for the extension's core functionality:

Permission Why it's needed What data is accessed
activeTab Access the content of the currently active tab when you click "Save" to add it to your knowledge graph Text content and URL of the active tab, only when you trigger the save action
tabs Access the URL and title of the current tab to capture page metadata (source URL, page title) for knowledge graph nodes URL and title of tabs — used only when saving a page to your knowledge graph
storage Store your authentication token and extension preferences locally on your device Auth token and user settings — no sensitive data beyond what's needed to keep you signed in
sidePanel Display DeepTrail's knowledge graph UI in Chrome's side panel No data is accessed through this permission — it only enables the side panel UI
alarms Schedule periodic background tasks such as token refresh or deferred processing. This permission does not access any user data — it only triggers timed events within the extension's service worker. No user data is accessed through this permission

Host Permissions

DeepTrail declares a host_permissions entry in its manifest for https://deep-trial.onrender.com/* — our backend API server hosted on Render. This permission is required so the extension can send saved page content to the backend for processing, receive knowledge graph data, and perform search queries. This is the only external domain the extension communicates with directly. All communication uses HTTPS.

Remote Code

DeepTrail does not use remote code. All extension code is bundled locally with the extension package. No scripts are loaded from external servers at runtime.

We follow the principle of minimal permissions — DeepTrail only requests the permissions strictly necessary for its functionality. We do not request broad permissions like <all_urls>, history, cookies, or webRequest.

8. Your Rights and Controls

You have the following rights regarding your data:

  • Access: Your entire knowledge graph is visible in the extension's side panel UI. You can see all nodes, connections, and metadata at any time.
  • Deletion (individual): You can delete individual nodes from your knowledge graph at any time through the extension UI.
  • Deletion (full account): You can request complete deletion of all your data by emailing tenthinlayofficial@gmail.com. We will permanently delete all your data from our servers within 30 days.
  • Data portability / export: You can request an export of your data by contacting us at the email above. (An in-app export feature is planned for a future release.)
  • Opt out: You can stop using DeepTrail and uninstall the extension at any time. No data is collected while the extension is inactive.
  • Rectification: If any data we hold about you is inaccurate, contact us and we will correct it.

To exercise any of these rights, contact us at tenthinlayofficial@gmail.com. You also have the right to lodge a complaint with your local data protection authority.
